🎯 Objetivo

Desenvolver um componente chamado "CollectionBundle" responsável por agrupar e exibir um conjunto de mídias relacionadas vídeos dentro de um canal.

A Collection Bundle deve funcionar como uma unidade de curadoria de conteúdo, permitindo organizar mídias por tema, facilitando a descoberta e melhorando a experiência de navegação do usuário.

📦 Contexto

A plataforma é focada em conteúdo e carreira, com estrutura baseada em canais. Cada canal pode conter múltiplas Collection Bundles, e cada bundle contém múltiplas mídias.

🎨 Formato do layout UI/UX

- O componente é um card de álbum clicável que leva à página via routerLink.
- Estrutura visual (de cima para baixo)
┌─────────────────────────────┐
│         board-cover         │  ← aspect-ratio 1:1 (quadrado)
│  ┌──────────────┬────────┐  │
│  │              │  img2  │  │  ← grid 2fr | 1fr
│  │    img1      ├────────┤  │
│  │  (cover-main)│  img3  │  │  ← cover-side (col direita, 2 imgs)
│  │              │        │  │
│  └──────────────┴────────┘  │
├─────────────────────────────┤
│  board-info                 │
│  Nome do canal (bold)       │  ← Digix
│  texto  (texto secundário)  │  ← Primeiros passos de como ultilizar a plataforma Habix
└─────────────────────────────┘
-Comportamento:
    - Hover: leve scale(1.02) + sublinhado no nome
    - Bordas arredondadas (--radius-lg) em todo o card
    - Link navegável (<a> com routerLink) para /:username/collection/:id
- Layout em formato de card ou bloco visual
- Responsivo (mobile-first)

🧠 Regras de Negócio

- Uma Collection Bundle pertence a um canal
- Deve ter fallback visual caso não haja mídia
- Deve permitir estado vazio (empty state)
