🎯 Objetivo

Atualizar a tela `/home` para suportar a renderização de dois tipos de cards:

1. Vídeos (já existentes via `app-pin-card-player-short`)
2. Collection Bundles (novo: `app-collection-bundle`)

Além disso, implementar navegação e mock de API para suportar os dados de Collection Bundle.

---

📦 Contexto

Atualmente, o componente `home-dynamic-interest-tabs` renderiza apenas cards de vídeo (`app-pin-card-player-short`).

Agora, ele deve suportar múltiplos tipos de conteúdo no feed, incluindo Collections (bundles de mídia).

---

🧩 Tarefas


### 1. pagina collection
Não é necessario criar a pagina collection

### 2. Navegação

* Ao clicar em um `app-collection-bundle`, redirecionar para:

```
/:username/collection/:id
```

* Implementar método:

```ts
onOpenCollection(bundle: CollectionBundle) {
  this.router.navigate([`/${bundle.username}/collection/${bundle.id}`]);
}
```

---

### 4. Misturar dados no feed da Home

* Combinar dados de vídeos + collections no mesmo array
* Garantir ordenação consistente (ex: intercalar ou por data futura)


---

### 5. Componente `app-collection-bundle`

* Deve aceitar:

```ts
@Input() bundle: CollectionBundle;
```

* Exibir:

  * Thumbnail (primeira mídia disponível ou fallback)
  * Descrição

* Estado vazio:

  * Caso não haja mídias, exibir placeholder

---

⚙️ Regras Técnicas

* Seguir padrão Angular atual do projeto
* Componentes desacoplados e reutilizáveis
* Evitar lógica pesada no template
* Tipagem forte (interfaces bem definidas)


---

📌 Resultado Esperado

A Home passa a exibir um feed híbrido (vídeos + collections), onde:

* Vídeos continuam sendo exibidos normalmente
* Collections aparecem como blocos especiais
* Clique em collection leva para `/username/collection/id`
* Dados vêm de mock integrado

Isso introduz um novo nível de organização de conteúdo e melhora a descoberta dentro da plataforma.
