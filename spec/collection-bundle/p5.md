# 🎯 Objetivo
Refatorar a página **Collection** para que ela abandone a aparência semelhante ao YouTube e passe a seguir **a mesma identidade visual da página Watch**, garantindo consistência visual entre as páginas da plataforma.

  `/home/azjob/workspace/app-pin/src/app/domain/collection/pages/collection`

- Página Watch (referência visual):
  `/home/azjob/workspace/app-pin/src/app/domain/watch`

# ❗ Problema atual
A página **Collection** apresenta características visuais semelhantes ao YouTube, como:
- Estrutura de layout
- Organização de conteúdo
- Estilo de exibição

Isso causa:
- Quebra de consistência com o restante da aplicação
- Falta de identidade própria
- Experiência fragmentada entre páginas

# ✅ Objetivo da refatoração
A página Collection deve:
- Adotar a **mesma linguagem visual da Watch**
- Manter sua funcionalidade original (listar coleções), mas com nova apresentação
- Seguir os mesmos padrões de:
  - Layout
  - Tipografia
  - Espaçamento
  - Hierarquia visual
  - Componentes reutilizáveis

# 🧩 Diretrizes de implementação

## 1. Layout
- Remover qualquer estrutura semelhante ao YouTube
- Reorganizar a página com base no layout da Watch
- Seguir o mesmo padrão de containers, grids e responsividade

## 2. Componentes visuais
- Reutilizar os mesmos componentes utilizados na Watch sempre que possível
- Padronizar:
  - Cards de conteúdo
  - Botões
  - Ícones
  - Badges
- Evitar criar novos componentes se já existirem equivalentes

## 3. Exibição das coleções
- Adaptar as coleções para o mesmo formato visual usado na Watch
- Garantir consistência na forma de apresentar:
  - Thumbnails
  - Títulos
  - Metadados

## 4. Interações
- Aplicar os mesmos padrões de interação da Watch:
  - Hover
  - Animações
  - Transições
  - Feedback visual

## 5. Estilo e identidade
- Eliminar qualquer referência visual ao YouTube:
  - Layout
  - Proporções
  - Organização de conteúdo
- Garantir que a Collection pareça uma extensão natural da Watch

## 6. Código e arquitetura
- Seguir boas práticas Angular:
  - Componentização
  - Reutilização
  - Separação de responsabilidades
- Se necessário, extrair componentes compartilhados para:
  `/shared/components`
- Evitar duplicação de estilos e lógica

# 🚀 Resultado esperado
Uma página Collection que:
- Seja visualmente consistente com a Watch
- Tenha identidade própria da plataforma
- Não remeta ao YouTube
- Seja escalável e reutilizável

# ⚠️ Importante
Não apenas replicar o visual, mas **adaptar a estrutura e o comportamento da página Collection ao sistema de design da Watch**.
