🎯 Objetivo

Desenvolver a página "Post" responsável por exibir um conteúdo aprofundado dentro da plataforma. 
Essa página é o principal ponto de conversão, onde o usuário consome um vídeo longo e é incentivado a avançar (ex: candidatura, treinamento, compra) através de um CTA "Saiba Mais".

---

🧠 Contexto do Produto

A página de Post é uma evolução da página de Collection.

- A Collection foca em uma experiência sequencial (playlist/trilha).
- A Page Post foca em um conteúdo principal + descoberta de conteúdos relacionados.

🏗️ Estrutura da Página

### 1. Player principal (topo/esquerda)
- Vídeo longo (media.long)
- Aspect ratio: 16:9
- Controles customizados (play, volume, progresso, fullscreen)
- Overlay com:
  - botão play
  - tempo do vídeo
- Autoplay opcional

---

### 2. Ações do conteúdo (abaixo do player)
- Like (com contador)
- Salvar
- Compartilhar
- Menu de opções (...)

---

### 3. Informações do Post
- Título (media.title)
- Views + tempo (views + timestamp)
- Descrição (media.description)
- Hashtags (media.slang)

---

### 4. CTA principal
- Botão: "Saiba Mais"
- Destaque visual (cor primária)
- Objetivo: conversão (vaga, produto, treinamento)
- Posicionamento: lado direito ou abaixo do conteúdo

---

### 5. Canal (autor do conteúdo)
- Avatar (channel.profilePicture)
- Nome (channel.profileName)
- Badge de verificado (channel.verified)
- Número de seguidores
- Botão: "Inscrever-se"

---

### 6. Comentários
- Lista paginada (comment.data)
- Exibir:
  - avatar
  - usuário
  - texto
  - likes
  - tempo
  - número de respostas
- Campo para novo comentário

---

### 7. Sidebar (lado direito)

⚠️ DIFERENÇA PRINCIPAL DA COLLECTION:

- NÃO exibir playlist
- Exibir conteúdos relacionados

#### Estrutura:
- Lista de cards verticais
- Cada item contém:
  - thumbnail
  - título
  - canal
- Scroll independente
- Sugestões baseadas no conteúdo atual

---

🔌 Integração com API

Endpoint:
GET /api/post/{id}

Criar método:

getPostById(id: string): Observable<Post>

Mapear resposta:

- media → player + conteúdo
- channel → autor
- comment → comentários
- métricas → likes, views, shares

---

📦 Modelo de Dados

Usar exatamente a estrutura fornecida:

- media.long → vídeo principal
- media.thumbnail → capa
- media.title → título
- media.description → descrição
- media.slang → hashtags
- channel → dados do criador
- comment.data → lista de comentários

---

🎨 UX / UI Guidelines

- Layout responsivo (desktop-first)
- Grid com duas colunas:
  - esquerda: conteúdo principal
  - direita: relacionados
- Sidebar fixa com scroll
- Botão "Saiba Mais" sempre visível (fixo ou destaque)
