# 🎯 Objetivo
Refatorar a página **Watch** para que ela deixe de ter aparência semelhante ao YouTube e passe a seguir **a mesma identidade visual da página Showcase**, garantindo consistência visual na aplicação.

# 📍 Caminhos envolvidos
- Página Watch (origem da refatoração):
  `/home/azjob/workspace/app-pin/src/app/domain/watch`

- Página Showcase (referência visual):
  `/home/azjob/workspace/app-pin/src/app/domain/showcase`

# ❗ Problema atual
A página **Watch** está com identidade visual muito próxima ao YouTube (layout, organização, estilos e comportamento), o que:
- Prejudica a identidade única da plataforma
- Cria inconsistência visual com o restante do sistema
- Diminui a percepção de produto próprio

# ✅ Objetivo da refatoração
A página Watch deve:
- Adotar **100% da linguagem visual da Showcase**
- Manter funcionalidade, mas alterar **estrutura e apresentação**
- Seguir os mesmos padrões de:
  - Espaçamento
  - Tipografia
  - Cores
  - Hierarquia visual
  - Componentização

# 🧩 Diretrizes de implementação

## 1. Layout
- Abandonar completamente o layout estilo YouTube (player + sidebar)
- Reorganizar a página seguindo o padrão estrutural da Showcase
- Utilizar a mesma lógica de grid, containers e responsividade

## 2. Componentes visuais
- Reutilizar componentes existentes da Showcase sempre que possível
- Caso necessário, extrair componentes reutilizáveis para:
  `/shared/components`
- Garantir consistência de:
  - Cards
  - Botões
  - Badges
  - Tipografia

## 3. Player de vídeo
- Integrar o player dentro do layout da Showcase (não como elemento dominante isolado)
- Tornar o player parte do fluxo visual (como um card ou bloco de conteúdo)

## 4. Interações
- Aplicar os mesmos padrões de interação da Showcase:
  - Hover
  - Animações
  - Transições
  - Feedback visual

## 5. Estilo e identidade
- Remover qualquer referência visual ao YouTube:
  - Layouts
  - Proporções
  - Organização
- Garantir que a Watch pareça uma extensão natural da Showcase

## 6. Código
- Seguir boas práticas Angular:
  - Componentização
  - Separação de responsabilidades
  - Reutilização
- Evitar duplicação de lógica/estilo
- Manter código limpo e escalável

# 🚀 Resultado esperado
Uma página Watch que:
- Seja visualmente consistente com a Showcase
- Tenha identidade própria da plataforma
- Não lembre o YouTube
- Seja moderna, reutilizável e escalável

# ⚠️ Importante
Não apenas "copiar visualmente", mas **adaptar estruturalmente** a Watch para o mesmo sistema de design da Showcase.
